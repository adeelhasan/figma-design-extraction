**Read this before starting. Follow the pipeline in order.**

## Pre-Flight

### 1. Verify MCP is Disabled
```
mcp__figma__* tools must NOT be used during extraction
```

### 2. Initialize Output Directory
```bash
python3 .claude/skills/figma-extraction/scripts/init-extraction.py
```
Creates timestamped `design-system-YYYYMMDD-HHMMSS/` directory with all subdirs and `design-system-latest` symlink. No need to check for existing output — timestamped dirs never collide.

## Data Architecture

### Python Scripts

All file operations use Python scripts — no inline curl/mkdir/bash needed.

| Script | Purpose | Phase |
|--------|---------|-------|
| `init-extraction.py` | Create timestamped dir + symlink | Pre-flight |
| `fetch-figma-file.py` | Fetch Figma API, cache response | Phase 1 |
| `export-images.py` | Download PNGs from Figma | Phase 1, 6 |
| `figma-query.py` | Query cached Figma data | Phase 1-6 |
| `file-ops.py` | File existence/count checks | Phase 7 |

All scripts live in `.claude/skills/figma-extraction/scripts/` and are invoked via `python3`.

### Figma Query Tool

All agents access Figma data through the query tool — **never read the full cache file directly**.

```bash
QUERY="python3 ${SCRIPTS}/figma-query.py --cache ${OUTPUT_DIR}/.cache/figma-file.json"

# Token extraction queries (~2-10KB each)
$QUERY colors          # unique solid fills with hex + context
$QUERY gradients       # gradient definitions (type, angle, stops)
$QUERY text-styles     # unique font/size/weight/lineHeight combos
$QUERY effects         # shadow + blur definitions with CSS
$QUERY radii           # corner radius values
$QUERY spacing         # auto-layout spacing values

# Screen extraction queries
$QUERY screen-layout "Dashboard"              # top-level sections (~2-15KB)
$QUERY section "Dashboard" "sidebar" --slim    # section subtree, slim (~2-70KB)
$QUERY section "Dashboard" "content/Widget-1" --slim  # nested sub-section (~7-15KB)

# Component queries
$QUERY components                             # all component definitions
$QUERY component "button/size/large/default"  # specific component subtree
```

### Essentials File

`figma-essentials.json` (~5KB) — compact table-of-contents generated by Phase 1. The orchestrator reads this to know screen/component lists. All agents can read it for structural context.

### Cache File

`.cache/figma-file.json` (~2.5MB) — full Figma API response. **Only the query tool reads this.** Agents must NOT read it directly (truncated by Read tool, overflows context).

## Pipeline

### Phase 1: Connect & Screenshot
```
AGENT: Task subagent
READ: prompts/connect.md
SCRIPTS: fetch-figma-file.py (API fetch), export-images.py (screenshots), figma-query.py (essentials)
OUTPUT: extraction-meta.json, .cache/figma-file.json, figma-essentials.json, preview/layouts/screenshots/*.png
```

### Phase 2: Fingerprint
```
AGENT: Task subagent
READ: prompts/design-system-fingerprint.md
DATA: screenshots (visual) + $QUERY colors + $QUERY gradients
OUTPUT: design-system-context.json
```

### Phase 3: Tokens (4 Parallel Tasks, haiku model)
```
AGENT: 4 parallel Task subagents
READ: prompts/extract-tokens.md (with tokenType = colors|typography|spacing|effects)
DATA: $QUERY colors/gradients/text-styles/effects/radii/spacing (per type)
OUTPUT: tokens/colors.css, tokens/typography.css, tokens/spacing.css, tokens/effects.css
```
Launch all 4 as parallel Task agents. Wait for all to complete.

### Phase 4: Components + Icons/Assets (2 Parallel Tasks)

Launch both in parallel:

**4a: Components**
```
AGENT: Task subagent
READ: prompts/extract-components.md
DATA: $QUERY components + $QUERY component "<name>"
OUTPUT: specs/components.md
```

**4b: Icons & Assets**
```
AGENT: Task subagent
READ: prompts/extract-icons.md -> OUTPUT: assets/icon-manifest.json
READ: prompts/extract-assets.md -> OUTPUT: assets/asset-manifest.json
SCRIPTS: export-images.py (image downloads)
DATA: $QUERY components
```

Wait for both to complete before proceeding.

### Phase 5: Screens (N Parallel Tasks)
```
AGENT: N parallel Task subagents (one per screen)
READ: prompts/extract-screen.md
DATA: $QUERY screen-layout + $QUERY section (incremental)
OUTPUT per screen:
  - specs/layouts/{Screen}.md
  - specs/layouts/{Screen}.json
  - preview/layouts/{Screen}.html
  - preview/layouts/data/{Screen}.json
```
Launch all as parallel Task agents. Wait for all to complete.

### Phase 6: Generate Index Files
```
AGENT: Task subagent (haiku)
OUTPUT:
  - specs/layouts.md — index
  - preview/layouts/index.html — gallery
  - preview/index.html — main dashboard
  - preview/tokens.html — token reference
```

### Phase 6b: Generate Design Reference
```
AGENT: Task subagent (haiku)
READ: prompts/generate-reference.md
DATA: design-system-context.json, tokens/*.css, specs/components.md
OUTPUT: design-reference.md
```

### Phase 7: Sanity Check

Verify all files exist using file-ops script:
```bash
python3 ${SCRIPTS}/file-ops.py exists \
  --files "tokens/colors.css,tokens/typography.css,tokens/spacing.css,tokens/effects.css,specs/components.md,design-system-context.json,extraction-meta.json,figma-essentials.json,design-reference.md" \
  --dir "${OUTPUT_DIR}"

python3 ${SCRIPTS}/file-ops.py compare \
  --pattern-a "specs/layouts/*.md" \
  --pattern-b "preview/layouts/*.html" \
  --exclude-b "index.html" \
  --dir "${OUTPUT_DIR}"
```

Open preview for human review:
```bash
open ${OUTPUT_DIR}/preview/index.html
```

## Rules

1. **READ prompt THEN EXECUTE** — don't skip reading
2. **ALL phases are Task subagents** — orchestrator dispatches, doesn't execute
3. **NEVER read the full cache** — use the query tool for all Figma data access
4. **Parallel where indicated** — tokens (4x) and screens (Nx) run concurrently
5. **Sequential otherwise** — fingerprint before tokens, tokens before screens
6. **No LLM verification** — sanity check is file-existence only
7. **Human reviews previews** — open HTML, don't ask LLM to validate
8. **Python scripts for all I/O** — no inline curl, mkdir, or bash file operations
